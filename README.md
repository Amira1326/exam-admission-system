# نظام التقييم والقبول — دليل الإعداد والنشر

✅ **قاعدة البيانات جاهزة ومُعدّة فعلياً** على مشروع Supabase باسم `exam-admission-system`،
والمفاتيح موجودة بالفعل داخل `supabase-config.js`. لا حاجة لإعادة الخطوتين 1 و2 بالأسفل.

## الملفات
- `index.html` — الواجهة الرئيسية (اختيار الفرع، تسجيل الطالب، تسجيل دخول الموظفين، لوحات التحكم)
- `exam.html` — صفحة أداء الاختبار (تجلب الأسئلة من القاعدة وتحفظ النتائج فيها)
- `app.js` — كل منطق الواجهة الرئيسية المتصل بقاعدة البيانات
- `supabase-config.js` — مكان وضع مفاتيح الاتصال بقاعدة بياناتك
- `schema.sql` — أوامر إنشاء قاعدة البيانات كاملة (جداول + صلاحيات + دوال)

## الخطوة 1: إنشاء قاعدة بيانات Supabase (مجانية)
1. اذهب إلى https://supabase.com وأنشئ حساب، ثم مشروع جديد (New Project).
2. من القائمة الجانبية اذهب إلى **SQL Editor** واختر **New query**.
3. انسخ محتوى ملف `schema.sql` كاملاً والصقه هناك، ثم اضغط **Run**.
   هذا سينشئ جميع الجداول والصلاحيات والدوال المطلوبة.

## الخطوة 2: ربط الملفات بقاعدة البيانات
1. من إعدادات مشروع Supabase اذهب إلى **Project Settings > API**.
2. انسخ قيمتي **Project URL** و **anon public key**.
3. افتح ملف `supabase-config.js` وضع القيمتين مكان:
   ```js
   const SUPABASE_URL = "PUT_YOUR_SUPABASE_URL_HERE";
   const SUPABASE_ANON_KEY = "PUT_YOUR_SUPABASE_ANON_KEY_HERE";
   ```

## الخطوة 3: إنشاء حسابات الموظفين (معلم / مسؤول قبول / مدير)
الطلاب يسجّلون نفسهم مباشرة من الموقع، لكن حسابات الموظفين تُنشأ يدوياً لأول مرة:
1. في Supabase اذهب إلى **Authentication > Users > Add user** وأنشئ حساباً ببريد وكلمة مرور (مثلاً admin@school.com).
2. انسخ الـ **User UID** الظاهر للحساب.
3. اذهب إلى **SQL Editor** ونفّذ:
   ```sql
   insert into profiles (id, name, role, branch)
   values ('USER-UID-هنا', 'اسم الموظف', 'admin', null);
   ```
   غيّر `role` إلى `teacher` أو `registrar` أو `admin` حسب الحساب.
4. كرر الخطوات لكل موظف تحتاجه (معلم، مسؤول قبول، مدير).

## الخطوة 4: إضافة أسئلة الاختبار
- سجّل دخولك كـ **مدير النظام**، وفي لوحته قسم "إدارة الأسئلة" أضف أسئلة الاختيار من متعدد لكل مادة ومرحلة.
- للأسئلة **الكتابية** (مثل بعض أسئلة اللغة العربية)، أضفها مباشرة عبر SQL Editor لأن نموذج الإضافة الحالي في الواجهة يدعم أسئلة الاختيار فقط:
  ```sql
  insert into questions (subject, grade, type, question_text, marks, instructions, order_index)
  values ('arabic','g1','written','اكتب الحرف المناسب للصورة', 6, 'حل السؤال على ورقة وارفع صورته', 1);
  ```
- لإضافة قطعة قراءة لمادة الإنجليزي:
  ```sql
  insert into passages (subject, grade, title, body, image_url)
  values ('english','g1','Sara in the Garden','Sara is in the garden...', null);
  ```

## الخطوة 5: التجربة محلياً قبل النشر
لأن المتصفحات تمنع بعض الطلبات عند فتح ملف HTML مباشرة (file://)، شغّل خادم محلي بسيط:
```bash
cd project
python3 -m http.server 8080
```
ثم افتح `http://localhost:8080` في المتصفح.

## الخطوة 6: النشر على GitHub Pages (مجاني)
1. أنشئ مستودع (Repository) جديد على GitHub وارفع محتوى مجلد `project` بالكامل إليه (index.html, exam.html, app.js, supabase-config.js).
   **لا** تحتاج لرفع `schema.sql` للموقع، هو فقط للإعداد الأولي في Supabase.
2. من إعدادات المستودع اذهب إلى **Settings > Pages**.
3. اختر Source: **Deploy from branch**، والفرع main، والمجلد `/root`.
4. بعد دقيقة سيظهر رابط موقعك بالشكل: `https://username.github.io/repo-name/`.

## ملاحظات مهمة
- **الأمان**: مفتاح `anon key` مصمم ليكون عاماً وآمناً للاستخدام في الواجهة الأمامية، بشرط أن تبقى صلاحيات RLS في `schema.sql` كما هي (تم إعدادها بحيث الطالب لا يرى بيانات غيره ولا يقدر يغيّر حالة قبوله بنفسه).
- **الرمز السري للطالب**: يُنشأ تلقائياً عند التسجيل (أول حرفين من الاسم + آخر 4 أرقام من الهوية) ويُستخدم مع رقم الهوية للدخول لصفحة الاختبار والتحقق من الحالة.
- **ميزات تجميلية** كانت في التصميم الأصلي (الشاتبوت، بطاقات المراجعة، وضع المراقب الوهمي، الأوسمة) تم إزالتها أو تبسيطها في هذه النسخة للتركيز على الوظائف الفعلية (تسجيل، قبول/رفض، اختبار حقيقي، تصحيح، تقارير). إذا كنت تريد إعادتها بشكل متصل بقاعدة البيانات فعلاً (وليس بيانات وهمية)، أخبرني وأضيفها.
- إذا احتجت صلاحيات أدق (مثل معلم يرى طلاب صفه فقط، أو مسؤول قبول يرى فرعه فقط)، يمكن تعديل سياسات RLS في `schema.sql` بسهولة — أخبرني وأجهزها لك.
